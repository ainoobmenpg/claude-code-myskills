## プロセス

1. すぐに仕様書を書かないでください。必要に応じて段階的に質問してください。
2. 少なくとも以下を明確にしてください:
   - 目的
   - ユーザー
   - 想定ユースケース
   - 入力と出力
   - 範囲内（in-scope）と範囲外（out-of-scope）
   - 制約条件
   - 受け入れ条件
   - リスクと未決定事項
3. 情報が不足している場合は、AskUserQuestionツールを使用してユーザーに直接質問してください。
4. 情報が収集できたら、仕様書の要約を日本語で作成してください。

## 下書き作成方法（重要）

仕様書の下書きを**章ごとに**Writeツールで書いてください。すべての章を一度に書こうとしないでください。

手順:
1. まず、Writeツールでspec-draft.mdに概要から機能リストまでを作成してください
2. 次に、Editツールで各機能の詳細を追加してください（機能が多い場合は複数ステップに分割）
3. 最後に、Editツールで共通制約と未決定事項を追加してください

各ステップ完了時にstatus.jsonを更新してください。

## ステータス管理

進捗に応じて以下のJSONを更新してください:
保存先: `{STATUS_FILE}`

### 質問中
```json
{
  "status": "waiting_for_user",
  "progress": "ユーザーの回答待ち（質問N）",
  "updated_at": "UTCタイムスタンプ"
}
```

### 下書き作成中
```json
{
  "status": "in_progress",
  "progress": "現在の進捗メッセージ（例: 概要から機能リストまで作成完了）",
  "updated_at": "UTCタイムスタンプ"
}
```

章を書くたびにprogressとupdated_atを更新してください。

### 完了時
```json
{
  "status": "completed",
  "progress": "仕様書下書き作成完了",
  "updated_at": "UTCタイムスタンプ"
}
```

### 完了時の手順

仕様書が完成したら、以下の順序で進めてください:
1. spec-draft.mdを保存してください（保存先: `{DRAFT_PATH}`）
2. status.jsonをcompletedに更新してください（上記JSONを参照）
3. 「仕様書下書きを保存しました」とのみ報告してください

### 失敗時
```json
{
  "status": "failed",
  "progress": "エラー内容",
  "updated_at": "UTCタイムスタンプ"
}
```
