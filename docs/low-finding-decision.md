# low が 1 件残ったときの判断表

low が残ったときに自動で「進めてよい」とは判断しません。影響の種類で切り分けます。

## 代表パターン

- 文言が少し曖昧
- 補助コメントや説明不足
- 将来の誤読につながる命名の粗さ
- 今すぐ壊れないが、証跡が薄い

## 進めてよい条件

- spec の acceptance は満たしている
- scope 外変更はない
- high / medium は 0
- low が「誤解しやすいが現挙動は正しい」種類に留まる
- 後続の verify で追加悪化していない

## 止める条件

- low の本文を読むと、実は acceptance 解釈ズレが含まれている
- low が複数の仕様解釈を生みそう
- docs task で literal な変更後文言が特定できていない
- low だが、次の人が同じ場所で迷うのが明らか

## 迷ったらどこを見るか

1. `spec.md` の `受け入れ条件`
2. `review.json` の low finding 本文
3. `verify.json` の acceptance 証跡
4. `checked_paths` / `checked_hunks`

## 実務上の判断

この repo の現行フローでは medium / low も基本は修正対象です。そこで「進めてよい」は、無視するという意味ではなく、「いったん完了に近いが、次の修正候補として残すか」を慎重に判断する、という意味で使ってください。
